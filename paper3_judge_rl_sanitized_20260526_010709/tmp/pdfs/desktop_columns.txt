Position Shortcut: How Reinforcement Learning Teaches LLM Judges to
Cheat
Anonymous ACL submission

=== PAGE 1 LEFT ===
Abstract
LLM judges are increasingly trained to evalu-
ate pairwise preferences and to provide reward
signals in post-training pipelines. These judges
are typically optimized and reported by accu-
racy, but accuracy is only meaningful if the gold
label is predicted from evidence about which
response better satisfies the prompt rather than
from artifacts of prompt construction. We
identify a simple but consequential failure of
this assumption: standard chosen/rejected
datasets place the preferred response first, so
a naive conversion to pairwise judge prompts
makes response position a perfect predictor of
the gold verdict. Trained judges can therefore
learn a position shortcut, achieving high ap-
parent accuracy by selecting the first response
rather than comparing the responses. We di-
agnose this failure with position-swap evalua-
tion across SFT, DPO, GRPO, reward designs,
label schemes, learning rates, and model fam-
ilies. On unbalanced data, training improves
accuracy while degrading position-swap con-
sistency; in the most extreme case, a judge
reaches 99% accuracy by almost always choos-
ing the first response. The shortcut persists
under composite rewards, prompt instructions,
relabeling, and model scaling. A simple data-
level fix—balancing preferred responses across
positions—removes the confound, restores con-
sistency, and preserves genuine accuracy gains.
Our results show that accuracy can conflate
judgment ability with positional exploitation,
motivating position-swap consistency as a stan-
dard diagnostic for RL-trained judges.
1 Introduction
LLM judges are increasingly trained to evaluate
pairwise preferences and to provide reward sig-
nals in post-training pipelines (Zheng et al., 2023;
Chiang et al., 2024). These judges are usually opti-
mized and reported by accuracy: whether their ver-
dict matches a gold preference label. This metric
carries an implicit assumption: the model should

This creates a basic question for trained judges:
when accuracy improves, has the model learned to
compare the responses, or has it learned a shortcut
in the training interface?

2 Related Work
Bias and Consistency in LLM Judges. Prior
work has shown that LLM judges are sensitive to
where candidate responses appear in a pairwise
prompt (Zheng et al., 2023; Wang et al., 2024;
Shi et al., 2024). This position bias interacts with
broader evaluator biases, including length prefer-
ence, self-preference, and reliance on surface fea-
tures (Wu and Aji, 2023; Koo et al., 2024; Ye et al.,
2024; Park et al., 2024; Singhal et al., 2024). Ex-
isting work therefore develops metrics and proto-
cols for measuring or mitigating inconsistent judg-
ments, such as swap-based consistency analysis,
BT-σ (Qian et al., 2026), inference-time consis-
tency checks (Wang et al., 2025), permutation-
aware RL objectives (Xu et al., 2025; Zheng et al.,
2026), and selective abstention for uncertain pair-
wise judgments (Badshah et al., 2026). These
studies establish that position sensitivity matters.
Our focus is the training-time source: standard
preference-data conversion can make response posi-
tion a rewarded feature before any evaluation-time
correction is applied.
RL Training for Preference Judges. Preference
learning has long used human judgments to
train language models (Christiano et al., 2017;
Stiennon et al., 2020; Ouyang et al., 2022; Bai
et al., 2022), and recent work applies similar
optimization recipes to the judges themselves.
JudgeLRM (Chen et al., 2025) trains reason-
ing judges with GRPO (Shao et al., 2024);
FairJudge (Yang et al., 2026) combines SFT,
DPO (Rafailov et al., 2023), and GRPO with
consistency-oriented objectives. These approaches
treat reward design or algorithmic consistency

constraints as the main levers for improving judge
behavior. We instead ask whether the data layout
makes a non-semantic feature predictive of reward.
Under a position confound, even a correct label
reward can be maximized by selecting the first
response, so reported accuracy gains must be
checked against position-swap consistency.
Shortcut Learning and Reward Hacking.
Shortcut learning occurs when models exploit
spurious correlations instead of task-relevant
features (Geirhos et al., 2020; McCoy et al., 2019;
Tu et al., 2020). Reward hacking is a related RL
failure mode in which policies exploit defects in
the reward signal (Skalse et al., 2022; Gao et al.,
2023; Pan et al., 2024), motivating constrained
or multi-objective RL approaches (Moskovitz
et al., 2023). The judge-training setting studied
here is subtly different. The reward can be a
correct match to the gold label, but if the data
layout makes position perfectly correlated with
that label, the shortcut is as rewarding as genuine
comparison. This is why reward engineering alone
is insufficient: the causal error lies in the data
layout, not only in the scalar reward.
3 Diagnostic Framework
3.1 Pairwise Judge Training and the Position
Confound
We study LLM judges trained to evaluate pairwise
preferences. Given a query q and two candidate
responses, the judge produces a rationale, a verdict
v ∈ {A, B, tie}, and a confidence score c ∈ [0, 1].
The intended behavior is comparative judgment:
the verdict should follow evidence about which
response better satisfies the prompt, not incidental
artifacts of the prompt interface.
The artifact we study is response position. The
position confound arises when preference data are
converted into pairwise judge prompts. In standard
chosen/rejected datasets, the preferred response
is stored first. In RewardBench (Lambert et al.,
2024), this means the gold label is “A” for every
training instance after conversion to Response A/B
prompts. Position is therefore perfectly correlated
with reward: under an accuracy reward, selecting
Response A always receives maximum reward.
This makes apparent accuracy ambiguous. A
trained judge can improve by learning comparison-
relevant evidence, by learning the positional fea-
ture, or by mixing both. Standard evaluation on
the same ordering cannot separate these sources,

the same label. The position confound is therefore
not an artificial perturbation introduced by our eval-

construct judge-training prompts.

Second, we remove the confound at the data

orientations of each pair: the original instance and

the dataset (Appendix K). Once position A is cor-

position cannot exceed chance accuracy; any im-

come from a global first-position shortcut.

3.3 Position-Swap Evaluation
We evaluate each model on 449 held-out test in-
stances, generating responses with temperature
0.1 and max 512 new tokens. For each instance,
we generate judgments on both the original and
position-swapped order. This creates a direct test of
whether the judge’s verdict follows the compared
responses or their positions.
We report three metrics:
• Accuracy: agreement with gold labels on the
evaluated order.
• Position consistency: the fraction of in-
stances where the verdict is logically equiva-
lent under position swap. If the model predicts
A on the original order, it should predict B on
the swapped order, and vice versa. Ties are
self-consistent.
• Position bias: the percentage-point gap be-
tween always-first and always-second incon-
sistent predictions.
Position consistency is the critical metric be-
cause it distinguishes position-invariant judgment
from position-dependent shortcut use. We treat ties
as self-consistent because a tie verdict is unchanged
by swapping response order. Tie rates are below
3% across all models and conditions; treating ties
as inconsistent changes reported consistency by
less than 1 pp and does not affect any conclusions.

4.3 Robustness Controls
We next test whether the shortcut is merely an
artifact of training duration, inference procedure,
prompt wording, or label tokens.
Training dynamics. The shortcut emerges as a
phase transition (Figure 4; full checkpoint curves in
Appendix I). Steps 0–100: genuine content learning
dominates (accuracy up, consistency stable). Steps
100–300: the shortcut overtakes content learning
(accuracy continues rising, consistency collapses).
Steps 300–500: the shortcut saturates. The com-
posite reward follows the same trajectory, confirm-
ing that all reward components are simultaneously
satisfiable by the shortcut. This temporal pattern
means early stopping could partially mitigate the
problem by catching the model before the short-
cut fully activates, but the window is narrow and
model-dependent. Only balanced data eliminates
the shortcut regardless of training duration.
Inference and prompt controls. Evaluation-
time swap filtering detects the shortcut but covers
only 60% of test pairs (SFT: 0% coverage). Bal-
anced training achieves full coverage with compa-
rable accuracy. The filtering approach discards the
very instances where the shortcut is most active,
which are exactly the instances where a genuine
judge would be most informative.
Prompt-level mitigation (“evaluate based on con-
tent quality only; ordering is random”) raises con-
sistency by less than 2 pp on Qwen2.5 and produces
zero improvement on Mistral. The shortcut is en-
coded in the model’s weights, not its instruction-
following behavior. RL training has reshaped in-
ternal representations to route through positional
features regardless of what the prompt says. The
problem must be addressed during training.

Label controls. One might hypothesize that the
shortcut merely memorizes the token “A” as a
reward-correlated surface feature. If so, changing
the label scheme during evaluation should disrupt it.
We test this by evaluating unbalanced-GRPO mod-
els with alternative label schemes: “Response 1/2”
and “Response Left/Right” (Table 3). The short-

and both model families. Zero parse failures con-
firm both models correctly follow the relabeled for-
mat. They output “[[1]]” or “[[Left]]” as instructed
rather than reverting to “[[A]].”
The model has learned that the structurally first-
listed response in the prompt correlates with re-
ward, and it applies this heuristic regardless of
how that position is labeled. The severity gradient
reappears across models: Mistral’s weaker content
representations offer even less competition to the
positional feature, resulting in near-complete domi-
nation of first-position selection. Qwen3-8B shows
the mildest form of the shortcut (88% first-position
selection vs 99% for Mistral), consistent with its
stronger baseline judge capability providing more
competition to the positional feature. This cross-
model replication shows that the mechanism is not
tied to a particular label token or model family.

5.2 Decomposing Apparent Accuracy
Balanced training provides a counterfactual for sep-
arating genuine judgment gains from shortcut gains.
We define genuine gain as balanced accuracy minus
the untrained baseline, and shortcut gain as unbal-
anced accuracy minus balanced accuracy. This
decomposition is conservative: it attributes to the
shortcut only the accuracy that disappears when the
position–reward correlation is removed.
Table 4 shows that the shortcut contribution
is substantial but model-dependent. In the pri-
mary Qwen2.5-7B setting, 11.0 percentage points
of apparent accuracy come from the shortcut.
For Mistral-7B, more than half of the apparent
33 pp gain is positional. Qwen3-8B shows a
much smaller shortcut component, consistent with
stronger comparison ability competing more effec-
tively against the positional feature.
5.3 External Check on Published RL-Trained
Judges
As an external check, we evaluate JudgeLRM’s
publicly released checkpoints (Chen et al., 2025)
with the same position-swap diagnostic (Table 5).
JudgeLRM-7B achieves 81.3% accuracy with
75.7% consistency, while JudgeLRM-3B achieves
76.8% accuracy with 67.9% consistency. In both
cases, the A-selection rate equals accuracy, indi-
cating that correct predictions are strongly aligned
with first-position preference on this evaluation.
These results do not imply that all gains in prior
judge training are shortcuts. They show that stan-
dard accuracy can hide a position-dependent com-
ponent, and that the gap is larger for the smaller
checkpoint. Position-swap consistency is therefore
necessary for interpreting reported gains from RL-
trained judges.

6 Discussion
The position shortcut highlights a failure mode that
is easy to miss when judge training is evaluated
only by accuracy. The reward need not be noisy or
misspecified: even a correct label reward can rein-
force the wrong feature when the data layout makes
that feature perfectly predictive. In this sense, the
failure is related to reward hacking (Skalse et al.,
2022; Gao et al., 2023) but not identical to it. The
policy is not exploiting a flawed reward model;
it is exploiting a confounded route to the correct
reward.
This distinction matters for mitigation. Adding
reward terms can only help if those terms make
the shortcut costly. In our setting, decisiveness and
calibration remain compatible with always choos-
ing the rewarded position, so they do not distin-
guish comparison-based judgments from positional
ones. Data balancing removes the confound be-
fore optimization by making each position correct
equally often. Algorithm-level methods such as
EIS-GRPO (Xu et al., 2025) and PA-GRPO (Zheng
et al., 2026), as well as evaluation-time methods
such as SCOPE (Badshah et al., 2026), are com-
plementary: they can enforce or test permutation
awareness, while data deconfounding removes the
simplest positional reward path at the source.
The practical implication is straightforward.
Pairwise judge training should randomize or bal-
ance response order before any supervised, pref-
erence, or RL objective is applied, and reported
accuracy should be paired with position-swap
consistency. This recommendation is not spe-
cific to RewardBench. Any pipeline that con-
verts chosen/rejected data into fixed-order judge
prompts can introduce the same confound unless it

are increasingly used as both evaluators and reward

7 Conclusion
We showed that accuracy is not, by itself, a certifi-
cate of judgment ability for trained LLM judges.
When preference data are converted into fixed-
order pairwise prompts, response position can be-
come a perfectly reward-predictive artifact. Stan-
dard optimization can then improve apparent accu-
racy by exploiting the training interface rather than
comparing the responses. This position shortcut
appears across SFT, DPO, and GRPO, persists un-
der reward engineering, prompt instructions, label
changes, and learning-rate controls, and becomes
weaker but not absent in models with stronger com-
parison ability.
The remedy is simple but diagnostic: remove the
position–label correlation before optimization. Bal-
anced response positions cancel the shortcut gra-
dient, restore position-swap consistency, and pre-
serve genuine accuracy gains. More broadly, our
results suggest that RL-trained judge evaluations
should report not only accuracy but also whether
the verdict is invariant to swapping response or-
der. For pairwise judges, accuracy should count
as evidence of judgment only when it survives the
comparison-interface change.
Limitations
Our experiments use 7–9B parameter open-source
models from three families (Qwen2.5, Qwen3, Mis-
tral) with LoRA training. We have not tested
frontier closed-source models or full fine-tuning,
though we expect the confound mechanism to ap-

through the data and reward signal. We validate
primarily on RewardBench, the standard judge eval-
uation benchmark. The position confound is struc-
tural to the chosen/rejected data format used
across community datasets (HH-RLHF, UltraFeed-
back), so the mechanism generalizes beyond our
specific test set, but empirical validation on addi-
tional evaluation suites would strengthen external
validity. We do not compare against permutation-
aware RL methods (EIS-GRPO, PA-GRPO) which
address position bias at the algorithm level rather
than the data level. Position-swap consistency mea-

B Data Construction
RewardBench (Lambert et al., 2024) provides
2,985 preference pairs with gold labels across four
domains: chat, reasoning, safety, and coding. We
split into 2,089 training, 447 validation (used for
early stopping and hyperparameter selection), and
449 test instances using a 70/15/15 stratified split.
In the original data, the preferred response occupies
position A for 100% of instances in all splits.
For unbalanced training, we use this data as-is.
For balanced training, we include both orientations
of each training pair: the original instance with
gold label A and a swapped mirror with gold label
B. This yields 4,178 balanced training prompts
and ensures that position A is preferred in exactly
half the training instances. We also run a non-
duplicated balanced control with 2,089 prompts in
Appendix K.

SFT 100% accuracy sanity check. The SFT

position confound operating through supervised

positional regularity in the gold outputs. The ra-

the verdict labels themselves are the confound. Bal-

“[[B]]” targets, the model must learn content-based
judgment to predict the correct verdict.

and position-swapped version of each test instance,
using the swap rate as our consistency metric.

L Threshold Effect Analysis
The learning rate sweep shows that shortcut activa-
tion exhibits threshold behavior rather than gradual
degradation (Figure 2). At low learning rates, the
model’s pre-trained attention patterns for content
understanding are not significantly disrupted, and
position-based features remain dormant. Once opti-
mization pressure crosses a critical point, the much
simpler position feature (one token at one location)
receives disproportionate gradient signal compared
to the distributed content features. The model’s
internal representation reorganizes around position
rather than content.
Cross-model validation on Qwen3-8B confirms
this interpretation. A stronger model with more

to activate. Practitioners using newer, stronger
