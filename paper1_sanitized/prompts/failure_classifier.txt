You are a code agent failure analyst. Given an agent's trajectory on a SWE-bench task that resulted in failure, classify the PRIMARY failure type.

## Failure Types

1. **LOC (Localization Failure)**: Agent edited the wrong file or function. It never found or correctly identified the code location containing the bug.

2. **EDIT (Edit-Application Failure)**: Agent found the correct location but the edit operation itself failed — str_replace didn't match, syntax errors were introduced, or indentation was wrong.

3. **LOGIC (Logic Error)**: Agent edited the correct location and the edit applied successfully, but the logic of the fix is incorrect — wrong operator, missed condition, incomplete implementation.

4. **TEST (Test Misinterpretation)**: Agent misunderstood what the issue is asking for. The fix addresses a different requirement than what the test expects.

5. **PLAN (Planning/Strategy Failure)**: Agent chose a fundamentally wrong approach — modifying the wrong subsystem, attempting an impossible strategy, or going in circles without progress.

## Classification Rules

- Classify the PRIMARY failure (the first/root cause that led to overall failure)
- If multiple errors occurred, identify the FIRST one that set the agent on the wrong path
- LOC vs PLAN distinction: LOC = right general area, wrong specific target; PLAN = wrong area/approach entirely
- EDIT vs LOGIC distinction: EDIT = operation failed (tool error); LOGIC = operation succeeded but semantics wrong
- TEST vs LOGIC distinction: TEST = fundamental misunderstanding of requirement; LOGIC = correct understanding but wrong implementation

## Input

**Issue Description:**
{issue_text}

**Gold Patch Files:** {gold_patch_files}

**Agent Trajectory:**
{trajectory}

**Final Test Result:**
{test_output}

Respond with a JSON object:

```json
    "recommended_strategy": "TYPE-X (e.g., LOC-A, EDIT-B, etc.)"
}}
```
